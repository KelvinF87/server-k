Manual de la API de Usuarios

Introducción

Este manual describe cómo usar la API de usuarios, que permite realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) sobre una lista de usuarios.

Endpoints

1. Obtener Lista de Usuarios

Endpoint: GET /api/users

Descripción: Devuelve una lista de todos los usuarios.

Ejemplo de Respuesta:

{
    "message": "List of users",
    "data": [
        {
            "id": 1,
            "name": "John Doe",
            "email": "john@example.com",
            "password": "password123"
        }
    ]
}

2. Añadir un Nuevo Usuario

Endpoint: POST /api/users

Descripción: Añade un nuevo usuario a la lista.

Ejemplo de Solicitud:

{
    "name": "Jane Doe",
    "email": "jane@example.com",
    "password": "password456"
}

Ejemplo de Respuesta:

{
    "message": "User added",
    "data": [
        {
            "id": 1,
            "name": "John Doe",
            "email": "john@example.com",
            "password": "password123"
        },
        {
            "id": 2,
            "name": "Jane Doe",
            "email": "jane@example.com",
            "password": "password456"
        }
    ]
}

3. Actualizar un Usuario

Endpoint: PUT /api/users/:id

Descripción: Actualiza la información de un usuario existente.

Ejemplo de Solicitud:

{
    "name": "Jane Smith",
    "email": "jane.smith@example.com",
    "password": "newpassword"
}

Ejemplo de Respuesta:

{
    "message": "User updated",
    "data": [
        {
            "id": 1,
            "name": "John Doe",
            "email": "john@example.com",
            "password": "password123"
        },
        {
            "id": 2,
            "name": "Jane Smith",
            "email": "jane.smith@example.com",
            "password": "newpassword"
        }
    ]
}

4. Eliminar un Usuario

Endpoint: DELETE /api/users/:id

Descripción: Elimina un usuario de la lista.

Ejemplo de Respuesta:

{
    "message": "User deleted",
    "data": [
        {
            "id": 1,
            "name": "John Doe",
            "email": "john@example.com",
            "password": "password123"
        }
    ]
}
